646 =head1 NAME
647 
648 Biodiverse::Index - Methods to build, access and control an index of elements
649 in a Biodiverse::BaseStruct object.
650 
651 =head1 SYNOPSIS
652 
653  use Biodiverse::Index;
654 
655 =head1 DESCRIPTION
656 
657 Store an index to the values contained in the containing object. This is used
658 to reduce the processing time for sppatial operations, as only the relevant
659 index elements need be searched for neighbouring elements, as opposed to the
660 entire file each iteration.
661 The index is ordered using aggregated element keys, so it is a "flat" index.
662 The index keys are calculated using numerical aggregation.
663 The indexed elements are stored as hash tables within each index element.
664 This saves on memory, as perl indexes hash keys globally.
665 Storing them as an array would increase the memory burder substantially
666 for large files.
667 
668 
669 =head2 Assumptions
670 
671 Assumes C<Biodiverse::Common> is in the @ISA list.
672 
673 Almost all methods in the Biodiverse library use {keyword => value} pairs as a policy.
674 This means some of the methods may appear to contain unnecessary arguments,
675 but it makes everything else more consistent.
676 
677 List methods return a list in list context, and a reference to that list
678 in scalar context.
679 
680 CHANGES NEEDED
681 
682 =head1 Methods
683 
684 These assume you have declared an object called $self of a type that
685 inherits these methods, normally:
686 
687 =over 4
688 
689 =item $self = Biodiverse::BaseStruct->new;
690 
691 =back
692 
693 
694 =over 5
695 
696 =item $self->build_index ('contains' => 4);
697 
698 Builds the index.
699 
700 The C<contains> argument is the average number of base
701 elements to be contained in each index key, and controls the resolutions
702 of the index axes. If not specified then the default is 4.
703 
704 Specifying different values may (or may not) speed up your processing.
705 
706 =item $self->delete_index;
707 
708 Deletes the index.
709 
710 =item $self->get_index_elements (element => $key, 'offset' => $offset);
711 
712 Gets the elements contained in the index key $element as a hash.
713 
714 If C<offset> is specified then it calculates an offset index key and
715 returns its contents. If this does not exist then you will get C<undef>.
716 
717 =item $self->get_index_elements_as_array (element => $key, 'offset' => $offset);
718 
719 Gets the elements contained in the index key $element as an array.
720 Actually calls C<get_index_elements> and returns the keys.
721 
722 =item $self->get_index_keys;
723 
724 Returns an array of the index keys.
725 
726 =item $self->index_element_exists (element => $element);
727 
728 Returns 1 if C<element> is specified and exists. Returns 0 otherwise.
729 
730 =item $self->snap_to_index (element => $element);
731 
732 Returns the container index key for an element.
733 
734 
735 =back
